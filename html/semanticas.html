Elementos Semánticos en HTML

Muchos sitios web contienen código HTML como: <div id="nav"> <div class="header"> <div id="footer"> para indicar la navegación, el encabezado y el pie de página. En HTML, hay algunos elementos semánticos que se pueden utilizar para definir diferentes partes de una página web:

Elemento HTML <section>

El elemento <section> define una sección en un documento. Según la documentación HTML de W3C, "una sección es un grupo temático de contenido, típicamente con un encabezado". Algunos ejemplos de dónde se puede usar un elemento <section> son capítulos, introducción, elementos de noticias y datos de contacto. Una página web podría dividirse normalmente en secciones para introducción, contenido y datos de contacto.

Elemento HTML <article>

El elemento <article> especifica contenido independiente y autocontenido. Un artículo debería tener sentido por sí mismo y ser posible distribuirlo de manera independiente del resto del sitio web. Ejemplos de dónde se puede usar el elemento <article> son publicaciones de foros, entradas de blogs, comentarios de usuarios, tarjetas de productos y artículos de periódicos.

Elemento HTML <header>

El elemento <header> representa un contenedor para contenido introductorio o un conjunto de enlaces de navegación. Un elemento <header> típicamente contiene uno o más elementos de encabezado (<h1> - <h6>), un logotipo o icono, y la información de autoría. Nota: Puedes tener varios elementos <header> en un documento HTML, pero <header> no puede colocarse dentro de un <footer>, <address> u otro elemento <header>.

Elemento HTML <footer>

El elemento <footer> define un pie de página para un documento o sección. Un elemento <footer> típicamente contiene información de autoría, información de derechos de autor, información de contacto, un mapa del sitio, enlaces de regreso arriba y documentos relacionados. Puedes tener varios elementos <footer> en un documento.

Elemento HTML <nav>

El elemento <nav> define un conjunto de enlaces de navegación. Ten en cuenta que no todos los enlaces de un documento deben estar dentro de un <nav>. El elemento <nav> está destinado solo para bloques principales de enlaces de navegación. Los navegadores, como lectores de pantalla para usuarios discapacitados, pueden utilizar este elemento para determinar si omitir la representación inicial de este contenido.

Elemento HTML <aside>

El elemento <aside> define algún contenido aparte del contenido en el que se encuentra, como una barra lateral. El contenido de <aside> debe estar indirectamente relacionado con el contenido circundante.
